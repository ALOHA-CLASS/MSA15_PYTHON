# 2024년 3회-2 - `type()` 비교 & 반환값

## 원본 코드

```python
def func(value):
  if type(value) == type(100):
    return 100
  elif type(value) == type(""):
    return len(value)
  else:
    return 20

a = '100.0'
b = 100.0
c = (100, 200)

print(func(a) + func(b) + func(c))
```

---

## 문법 해설

### `type()` 함수

```python
type(100)    # <class 'int'>
type("")     # <class 'str'>
type(100.0)  # <class 'float'>
type(())     # <class 'tuple'>
```

- 인수의 **자료형(타입)** 을 반환
- `type(값1) == type(값2)` : 두 값의 타입이 같은지 비교

---

### isinstance() vs type() 비교

```python
# 이 코드 방식 (type 직접 비교)
type(value) == type(100)    # int 타입인지 확인

# 더 파이썬스러운 방법
isinstance(value, int)
```

---

### 각 변수의 타입 확인

| 변수 | 값 | 타입 |
|------|-----|------|
| `a` | `'100.0'` | `str` |
| `b` | `100.0` | `float` |
| `c` | `(100, 200)` | `tuple` |

---

### func 호출 분석

```python
func(a)   # a = '100.0' (str)
```
- `type('100.0') == type(100)` → `str == int` → **False**
- `type('100.0') == type("")`  → `str == str` → **True**
- → `return len('100.0')` = **5**

```python
func(b)   # b = 100.0 (float)
```
- `type(100.0) == type(100)` → `float == int` → **False**
- `type(100.0) == type("")`  → `float == str` → **False**
- → `return 20` (else)

```python
func(c)   # c = (100, 200) (tuple)
```
- `type((100,200)) == type(100)` → `tuple == int` → **False**
- `type((100,200)) == type("")`  → `tuple == str` → **False**
- → `return 20` (else)

---

### `len()` 함수

```python
len('100.0')   # 5  ← '1','0','0','.','0' 총 5글자
```

## 최종 계산

```python
func(a) + func(b) + func(c)
=  5    +   20   +   20
= 45
```
