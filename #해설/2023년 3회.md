# 2023년 3회 - 빈칸 채우기 문제 (`split`)

## 원본 코드 (빈칸 포함)

```python
print("파이썬 입출력에 대한 문제입니다.")
 
num1, num2 = input()._____()
num1 = int(num1)
num2 = int(num2)
print(num1, num2)
 
num3 = num1 + num2
print(num1 + " + " + num2 + " = " + num3)
```

---

## ✅ 정답

```python
num1, num2 = input().split()
```

---

## 문법 해설

### `input()` — 사용자 입력

```python
input()   # 콘솔에서 문자열 입력 받기 (반환 타입: str)
```

---

### `.split()` — 문자열 분리

```python
input().split()
```

- 입력 문자열을 **공백 기준**으로 나누어 **리스트** 반환
- `"10 20".split()` → `['10', '20']`

```python
num1, num2 = input().split()
# 입력: "10 20"
# → ['10', '20'] 언패킹 → num1='10', num2='20'
```

---

### 언패킹 (Unpacking)

```python
num1, num2 = ['10', '20']
# num1 = '10', num2 = '20'
```

- 리스트/튜플의 원소 수와 변수 수가 일치해야 함

---

### `int()` 타입 변환

```python
num1 = int(num1)   # '10' → 10
num2 = int(num2)   # '20' → 20
```

- `input()`은 항상 **문자열** 반환 → 숫자 연산 위해 `int()` 변환 필요

---

### ⚠️ 코드 오류 안내 (마지막 줄)

```python
# 오류 있는 코드
print(num1 + " + " + num2 + " = " + num3)
```

- `num1`, `num2`, `num3` 은 `int` 타입 → 문자열과 `+` 연산 불가 → `TypeError`
- 올바른 작성법:

```python
# 방법 1: f-string
print(f"{num1} + {num2} = {num3}")

# 방법 2: str() 변환
print(str(num1) + " + " + str(num2) + " = " + str(num3))

# 방법 3: print 다중 인수
print(num1, "+", num2, "=", num3)
```

---

## split() 추가 옵션

```python
"a,b,c".split(',')    # 구분자 지정 → ['a', 'b', 'c']
"a b c".split()       # 공백 기준 → ['a', 'b', 'c']
"a b c".split(' ', 1) # 최대 1번만 분리 → ['a', 'b c']
```
