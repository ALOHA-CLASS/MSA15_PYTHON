# 2020년 2회 - 세트(Set) 연산

## 원본 코드

```python
asia = {"한국", "중국", "일본"}
asia.add("베트남")
asia.add("중국")
asia.remove("일본")
asia.update(["홍콩", "한국", "태국"])
print(asia)
```

---

## 문법 해설

### 세트(Set) 자료형

```python
asia = {"한국", "중국", "일본"}
```

- `{}` 중괄호로 생성하지만 `딕셔너리`와 구분됨 (딕셔너리는 `key: value` 쌍)
- **특징**: 중복 불허, 순서 없음(unordered), 인덱싱 불가

---

### `.add(요소)` — 단일 요소 추가

```python
asia.add("베트남")  # {"한국", "중국", "일본", "베트남"}
asia.add("중국")    # "중국"은 이미 존재 → 무시됨 (변화 없음)
```

- 이미 존재하는 요소를 add해도 오류 없이 무시

---

### `.remove(요소)` — 요소 제거

```python
asia.remove("일본")  # {"한국", "중국", "베트남"}
```

- 해당 요소 삭제
- ⚠️ 존재하지 않는 요소 remove 시 → `KeyError` 발생
- 오류 없이 삭제하려면 `.discard()` 사용

---

### `.update(iterable)` — 여러 요소 일괄 추가

```python
asia.update(["홍콩", "한국", "태국"])
# "한국"은 이미 존재 → 무시
# 결과: {"한국", "중국", "베트남", "홍콩", "태국"}
```

- 리스트, 튜플, 세트 등 이터러블 전달 가능
- 중복 요소는 자동으로 무시

---

## 실행 흐름

| 단계 | 연산 | 세트 상태 |
|------|------|-----------|
| 초기 | `{"한국", "중국", "일본"}` | {한국, 중국, 일본} |
| add("베트남") | 추가 | {한국, 중국, 일본, 베트남} |
| add("중국") | 중복 → 무시 | {한국, 중국, 일본, 베트남} |
| remove("일본") | 삭제 | {한국, 중국, 베트남} |
| update([...]) | 홍콩·태국 추가, 한국 중복 무시 | {한국, 중국, 베트남, 홍콩, 태국} |

---

## 출력 결과

```
{'한국', '중국', '베트남', '홍콩', '태국'}
```

> ⚠️ 세트는 **순서가 없으므로** 실행마다 출력 순서가 다를 수 있음  
> 시험에서는 **포함된 요소**가 맞는지를 확인
